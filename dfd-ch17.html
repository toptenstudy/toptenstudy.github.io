10. Which of the following is not a guideline for drawing a data flow diagram (dfd)?


The guidelines that must be adhered to when drawing a DFD (Data Flow Diagram) are as follows :

Data cannot move back and forth between two entities.Data cannot move back and forth between two data storage.An entity cannot send data directly to a data store.A process must have at least one data flow for input and one for output.

Data Flow Diagram (DFD) is a diagram that describes the flow of data from a process or information system. In DFD, there is information related to the input and output of each of these processes. DFD also has various functions, such as conveying system design, describing systems, and designing models.

In the data flow diagram, there is no control over its flow, so there are no rules regarding decisions or repetition. The form of depiction is in the form of a data flowchart with a more specific scheme. According to Kenneth Kozar, the purpose of the DFD itself is to act as a provider or bridge between the user and the system.

The use of DFD (Data Flow Diagram) is widely used to assist application developers, especially in the process of making an information system. DFD was first popularized by Larry Constantine and Ed Yourdon in 1970.

The diagram was also the first time written in a classic text on SADT (Structured Analysis and Design Technique). The notation in the data flow diagram also refers to the graph theory which was originally used to model the workflow of an organization.
